# 记忆引擎 - 透明文献检索工作流程 v2.0

## 💡 背景与教训
- **时间**: 2026-03-29 
- **事件**: 文献检索Agent在`test_physics_msc.pdf`（纯物理课程目录）中虚构了大量心理学文献引用
- **影响**: 系统设计可能建立在虚假的学术基础上
- **教训**: 必须建立透明、可验证的工作流程，禁止虚构文献

## 🔄 五阶段工作流程

### 阶段1: 需求分析
```
目标: 明确真实文献需求
输入: 记忆引擎各模块设计文档
输出: topics_needed.json
关键: 基于实际理论模型，而非虚构概念
```

### 阶段2: 平台评估  
```
目标: 评估数据源可靠性
输入: 可用数据库列表
输出: platform_assessment.md
关键: 优先使用arXiv（开放获取），次选学校订阅资源
```

### 阶段3: 真实搜索执行
```
目标: 执行实际搜索并下载文献
输入: 关键词、搜索配置
输出: search_results.json, downloaded_pdfs/
关键: 只记录实际找到的内容，空白结果也要如实报告
```

### 阶段4: 文献整理与验证
```
目标: 整理文献并验证相关性
输入: 下载的PDF文件
输出: literature_summary.md, verification_log.txt  
关键: 每篇文献都必须有本地PDF文件用于验证
```

### 阶段5: 质量与透明度审计
```
目标: 确保工作流程的透明度
输入: 所有阶段输出
输出: audit_report.md, verification_checklist.txt
关键: 用户可随时审计任何引用
```

## 📁 文件组织结构

```
memory_engine/literature/
├── 📄 README.md               # 本说明
├── 📂 pdfs/                   # 所有下载的PDF文件
├── 📂 reports/                # 文献综述报告
├── 📂 metadata/               # 搜索元数据（时间、参数、结果）
├── 📂 citations/              # 标准引用格式
└── 📂 audits/                 # 透明度审计记录
```

## 🚨 安全措施

1. **禁止虚构**: 任何检索失败都必须直接报告，不得编造结果
2. **真实存储**: 所有PDF文件都必须真实保存在本地供验证
3. **详细记录**: 搜索参数、时间戳、结果数量必须详细记录
4. **可验证性**: 用户可以随时查看原始PDF验证任何引用

## 🧪 验证清单

每个文献检索任务完成后，必须检查：

- [ ] 是否有真实的PDF文件保存在`pdfs/`文件夹？
- [ ] 搜索参数是否正确记录在`metadata/`中？
- [ ] 文献综述是否基于真实的PDF内容？
- [ ] 所有引用是否都可追溯到具体的PDF文件？
- [ ] 是否有任何虚构或编造的内容？
