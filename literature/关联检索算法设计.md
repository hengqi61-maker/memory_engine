# 关联检索算法设计文档

## 1. 扩散激活算法详细设计

### 1.1 算法原理

扩散激活模拟人类记忆检索中的联想过程：
1. **初始化**: 查询节点获得初始激活值
2. **传播**: 激活值沿关联边向邻居节点传播
3. **衰减**: 传播过程中激活值按距离和边权重衰减
4. **收集**: 收集达到阈值的节点作为检索结果

### 1.2 数学公式

**激活传播公式**:
```
A_j(t+1) = ∑_{i∈N(j)} A_i(t) * w_ij * γ^d
```
其中：
- `A_j(t+1)`: 节点j在t+1时刻的激活值
- `A_i(t)`: 邻居节点i在t时刻的激活值
- `w_ij`: 边i→j的权重 (0-1)
- `γ`: 衰减因子 (默认0.7)
- `d`: 从查询节点到当前节点的距离

**归一化**:
```
A_j = A_j / max_activation  # 归一化到[0,1]
```

### 1.3 伪代码实现

```python
def spreading_activation(graph, query_node_ids, max_depth=3, threshold=0.1):
    """
    扩散激活算法
    
    Args:
        graph: 记忆关联图
        query_node_ids: 查询节点ID列表
        max_depth: 最大传播深度
        threshold: 激活阈值
    
    Returns:
        Dict[node_id: activation_value]
    """
    # 初始化激活值
    activations = {node_id: 1.0 for node_id in query_node_ids}
    visited = set(query_node_ids)
    
    # 按层传播
    for depth in range(max_depth):
        new_activations = {}
        
        for node_id, activation in activations.items():
            if activation < threshold:
                continue
                
            # 获取邻居
            neighbors = graph.get_neighbors(node_id)
            
            for neighbor_id, edge_weight in neighbors:
                if neighbor_id in visited:
                    continue
                    
                # 计算传播后的激活值
                decay_factor = DECAY_FACTOR ** (depth + 1)
                propagated_activation = activation * edge_weight * decay_factor
                
                # 累加激活值
                if neighbor_id in new_activations:
                    new_activations[neighbor_id] += propagated_activation
                else:
                    new_activations[neighbor_id] = propagated_activation
        
        # 合并激活值
        for node_id, activation in new_activations.items():
            activations[node_id] = activation
            visited.add(node_id)
        
        # 归一化
        max_act = max(activations.values()) if activations else 1.0
        if max_act > 0:
            for node_id in activations:
                activations[node_id] /= max_act
    
    # 过滤低于阈值的节点
    results = {node_id: act for node_id, act in activations.items() 
               if act >= threshold and node_id not in query_node_ids}
    
    return results
```

### 1.4 优化策略

**性能优化**:
1. **早期剪枝**: 激活值低于阈值时停止传播
2. **并行传播**: 使用多线程传播不同分支
3. **缓存激活值**: 缓存常见查询的激活结果
4. **限制分支因子**: 每个节点只传播到top-k邻居

**质量优化**:
1. **竞争机制**: 高激活节点抑制相邻节点
2. **多样化传播**: 确保不同方向的均衡传播
3. **时间衰减**: 旧记忆的激活值额外衰减

## 2. 图遍历检索算法设计

### 2.1 广度优先搜索 (BFS) 检索

```python
def bfs_traversal(graph, start_node_id, max_hops=3, weight_threshold=0.3):
    """
    基于BFS的关联检索
    
    Returns:
        List[Tuple[node_id, relevance_score]]
    """
    from collections import deque
    
    visited = set([start_node_id])
    queue = deque([(start_node_id, 0, 1.0)])  # (node_id, hop, path_weight)
    results = []
    
    while queue:
        node_id, hop, path_weight = queue.popleft()
        
        if hop >= max_hops:
            continue
        
        # 获取邻居
        neighbors = graph.get_neighbors(node_id)
        
        for neighbor_id, edge_weight in neighbors:
            if neighbor_id in visited:
                continue
                
            if edge_weight < weight_threshold:
                continue
            
            # 计算到达该节点的累积权重
            new_path_weight = path_weight * edge_weight * HOP_DECAY
            
            # 记录结果（排除起始节点）
            if hop > 0:
                results.append((neighbor_id, new_path_weight))
            
            # 加入队列继续传播
            queue.append((neighbor_id, hop + 1, new_path_weight))
            visited.add(neighbor_id)
    
    # 按相关性排序
    results.sort(key=lambda x: x[1], reverse=True)
    return results[:MAX_RESULTS]
```

### 2.2 深度优先搜索 (DFS) 检索

```python
def dfs_traversal(graph, start_node_id, max_depth=3, weight_threshold=0.3):
    """
    基于DFS的关联检索，适合发现深度关联链
    """
    results = []
    
    def dfs(current_id, depth, path_weight, path):
        if depth >= max_depth:
            return
        
        if len(results) >= MAX_RESULTS:
            return
        
        neighbors = graph.get_neighbors(current_id)
        
        for neighbor_id, edge_weight in neighbors:
            if neighbor_id in path:  # 避免环路
                continue
                
            if edge_weight < weight_threshold:
                continue
            
            new_path_weight = path_weight * edge_weight * DEPTH_DECAY
            
            # 记录结果（排除起始节点）
            if depth >= 1:
                results.append({
                    'node_id': neighbor_id,
                    'score': new_path_weight,
                    'path': path + [neighbor_id],
                    'depth': depth + 1
                })
            
            # 递归搜索
            dfs(neighbor_id, depth + 1, new_path_weight, path + [neighbor_id])
    
    dfs(start_node_id, 0, 1.0, [start_node_id])
    
    # 按分数排序
    results.sort(key=lambda x: x['score'], reverse=True)
    return results
```

### 2.3 随机游走检索

```python
def random_walk_retrieval(graph, start_node_id, num_walks=100, walk_length=10):
    """
    随机游走检索，发现潜在的弱关联
    """
    from random import choice
    
    scores = {}
    
    for _ in range(num_walks):
        current_node = start_node_id
        path_weight = 1.0
        
        for step in range(walk_length):
            neighbors = graph.get_neighbors(current_node)
            if not neighbors:
                break
            
            # 按权重概率选择下一个节点
            neighbor_ids = [n[0] for n in neighbors]
            weights = [n[1] for n in neighbors]
            
            # 归一化权重作为概率
            total_weight = sum(weights)
            if total_weight == 0:
                break
                
            probabilities = [w/total_weight for w in weights]
            
            # 随机选择（可按概率分布）
            next_node = choice(neighbor_ids)  # 简化实现
            next_weight = dict(zip(neighbor_ids, weights))[next_node]
            
            # 更新路径权重
            path_weight *= next_weight
            
            # 跳过起始节点
            if step > 0:
                if next_node in scores:
                    scores[next_node] = max(scores[next_node], path_weight)
                else:
                    scores[next_node] = path_weight
            
            current_node = next_node
    
    # 转换为排序列表
    results = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    return results[:MAX_RESULTS]
```

## 3. 混合检索集成方案

### 3.1 分阶段混合检索架构

```python
class HybridRetrievalSystem:
    def __init__(self, components):
        self.vector_retriever = components['vector']
        self.spreading_activation = components['spreading']
        self.graph_traversal = components['traversal']
        self.config = components['config']
    
    def retrieve(self, query, top_k=10):
        # 阶段1: 向量快速初筛 (top-100)
        vector_results = self.vector_retriever.retrieve(
            query, top_k=self.config['stage1_top_k']
        )
        
        # 提取候选节点ID
        candidate_ids = [r.memory.id for r in vector_results]
        
        # 阶段2: 扩散激活精排
        activation_scores = self.spreading_activation.search(
            candidate_ids, 
            max_depth=self.config['activation_depth']
        )
        
        # 阶段3: 图遍历发现间接关联
        traversal_scores = {}
        for node_id in candidate_ids[:self.config['traversal_start_nodes']]:
            traversal_results = self.graph_traversal.bfs_traversal(
                node_id, max_hops=self.config['traversal_hops']
            )
            for tid, score in traversal_results:
                traversal_scores[tid] = max(traversal_scores.get(tid, 0), score)
        
        # 综合评分
        final_scores = self.combine_scores(
            vector_results, activation_scores, traversal_scores
        )
        
        # 阶段4: 重排与抑制
        reordered = self.rerank_with_inhibition(final_scores)
        
        return reordered[:top_k]
    
    def combine_scores(self, vector_results, activation_scores, traversal_scores):
        """综合各阶段评分"""
        combined = {}
        
        # 向量相似度分数
        for result in vector_results:
            mem_id = result.memory.id
            combined[mem_id] = {
                'vector': result.score,
                'activation': activation_scores.get(mem_id, 0),
                'traversal': traversal_scores.get(mem_id, 0),
                'total': 0.0
            }
        
        # 归一化各维度分数
        self.normalize_scores(combined)
        
        # 加权求和
        for mem_id, scores in combined.items():
            total = (
                self.config['weights']['vector'] * scores['vector'] +
                self.config['weights']['activation'] * scores['activation'] +
                self.config['weights']['traversal'] * scores['traversal']
            )
            scores['total'] = total
        
        return combined
    
    def normalize_scores(self, scores_dict):
        """归一化各维度分数到[0,1]"""
        dimensions = ['vector', 'activation', 'traversal']
        
        for dim in dimensions:
            values = [scores[dim] for scores in scores_dict.values()]
            if not values:
                continue
                
            max_val = max(values)
            min_val = min(values)
            
            if max_val > min_val:
                # 最小-最大归一化
                for scores in scores_dict.values():
                    scores[dim] = (scores[dim] - min_val) / (max_val - min_val)
            elif max_val > 0:
                # 所有值相同且非零，设为1
                for scores in scores_dict.values():
                    scores[dim] = 1.0
```

### 3.2 抑制机制实现

```python
class RetrievalInhibition:
    """检索抑制机制"""
    
    def __init__(self, config):
        self.config = config
        
    def apply_inhibition(self, scored_items):
        """
        应用抑制机制
        
        Args:
            scored_items: List[Tuple[memory_id, score]]
        
        Returns:
            抑制后的排序列表
        """
        if not scored_items:
            return []
        
        # 容量限制
        capacity = self.config.get('capacity', 7)
        if len(scored_items) > capacity:
            scored_items = scored_items[:capacity * 2]  # 保留更多以供抑制
        
        # 类型抑制：限制同类型记忆数量
        scored_items = self.type_based_inhibition(scored_items)
        
        # 主题抑制：确保主题多样性
        scored_items = self.topic_based_inhibition(scored_items)
        
        # 侧抑制：高分数抑制相似邻居
        scored_items = self.lateral_inhibition(scored_items)
        
        return scored_items[:capacity]
    
    def type_based_inhibition(self, scored_items):
        """基于类型的抑制"""
        type_counts = {}
        result = []
        
        for mem_id, score in scored_items:
            mem_type = self.get_memory_type(mem_id)
            
            # 检查该类型是否已超过限制
            type_limit = self.config.get('type_limits', {}).get(mem_type, 3)
            if type_counts.get(mem_type, 0) >= type_limit:
                continue  # 抑制该类型的新记忆
            
            result.append((mem_id, score))
            type_counts[mem_type] = type_counts.get(mem_type, 0) + 1
        
        return result
    
    def topic_based_inhibition(self, scored_items):
        """基于主题的多样性控制"""
        # 使用MMR (Maximal Marginal Relevance) 算法
        
        selected = []
        remaining = scored_items.copy()
        
        # 第一个选择最高分的
        if remaining:
            best_id, best_score = remaining.pop(0)
            selected.append((best_id, best_score))
        
        while remaining and len(selected) < self.config.get('max_diverse', 5):
            # 计算每个剩余项的边际相关性
            mmr_scores = []
            
            for mem_id, score in remaining:
                # 计算与已选记忆的最大相似度
                max_similarity = 0
                for sel_id, _ in selected:
                    sim = self.compute_similarity(mem_id, sel_id)
                    max_similarity = max(max_similarity, sim)
                
                # MMR公式：权衡相关性和多样性
                mmr = self.config['lambda'] * score - (1 - self.config['lambda']) * max_similarity
                mmr_scores.append((mem_id, score, mmr))
            
            # 选择MMR最高的
            mmr_scores.sort(key=lambda x: x[2], reverse=True)
            best_mmr = mmr_scores[0]
            
            selected.append((best_mmr[0], best_mmr[1]))
            remaining = [(mid, score) for mid, score, _ in mmr_scores[1:]]
        
        return selected
    
    def lateral_inhibition(self, scored_items):
        """侧抑制：高激活抑制相似邻居"""
        # 构建相似度矩阵
        items = scored_items[:20]  # 只处理前20个
        
        for i in range(len(items)):
            if items[i][1] < 0.5:  # 低激活节点不受影响
                continue
                
            for j in range(i + 1, len(items)):
                mem_id_i, score_i = items[i]
                mem_id_j, score_j = items[j]
                
                # 计算相似度
                sim = self.compute_similarity(mem_id_i, mem_id_j)
                
                if sim > self.config['inhibition_threshold']:
                    # 抑制分数较低的节点
                    if score_i > score_j:
                        # 按比例抑制
                        inhibition = sim * self.config['inhibition_strength']
                        items[j] = (mem_id_j, score_j * (1 - inhibition))
                    else:
                        items[i] = (mem_id_i, score_i * (1 - inhibition))
        
        # 重新排序
        items.sort(key=lambda x: x[1], reverse=True)
        return items
```

```python
engine.retrieve_enhanced(query)

# 新增纯关联检索方法
engine.retrieve_by_association(query_node_ids, method='spreading')
```

### 8.2 监控指标

关键监控指标:
1. **检索延迟**: 平均/95分位/99分位延迟

3. **图统计**: 节点数、边数、平均度数
4. **结果质量**: 人工评估或自动评估分数
5. **内存使用**: 图结构和缓存的内存占用
